# Verité 新闻验证机制

Verité 是一套面向用户上传文字、图片、视频、新闻链接或截图的事实验证机制。目标不是判断“看起来像不像真的”，而是把信息拆成可验证主张，分别从联网交叉检索、逻辑一致性、历史复盘、来源链溯源、现实世界旁证、统计异常与基准率、内容与媒介完整性七个角度打分，最后输出 0-100% 的可信度结论。

后验证复核机制不计入单次评分角度，而是作为独立机制，在 1 小时、24 小时、72 小时、7 天等节点重新检查新证据、撤稿、更正、官方回应和平台处置，并更新结论。

## 1. 核心原则

1. 先拆 claim，再判断真假。一个新闻标题里可能同时包含真实事实、推测和错误因果，必须拆开验证。
2. 区分“某人说过这句话”和“这句话描述的事实是真的”。认证名人账号可以证明发言存在，但不能自动证明发言内容为真。
3. 区分原始来源、转载来源和同源扩散。十家媒体转同一篇通讯社稿件，只算一个独立来源。
4. 权威媒体不等于 100% 真实。BBC、纽约时报、路透、彭博等来源可以给高起点，但如果文章只写“知情人士透露”或没有原始证据，需要降权。
5. 社交平台热度不等于事实。点赞、阅读、转发和名人互动只能提高“值得关注度”和少量可信度，不能替代证据链。
6. 现实世界中的重大事件通常会留下外部痕迹，例如官方记录、招标、航班、卫星图、工商变更、招聘、市场价格、医院或地方政府通报。
7. AI 检测、PS 检测和深度伪造检测不能单独定案，需要和元数据、反向图片搜索、地理定位、叙事语言、版本演化、来源链一起判断。
8. 最终输出必须给出百分比、证据、反证和可疑点，不使用“可能是真的”“感觉不太对”这类无法执行的判断。
9. 英语信息网络是全局验证层。无论用户输入中文、英文或其他语言，系统都必须在保留原语言检索的同时，生成英语概念、主体、动作、时间和领域词，进入新闻、官方、现实旁证、社交、自媒体、主动证伪和学术渠道共同检索。

### 1.1 开源机制借鉴

- Loki 风格拆分：把长文本、标题或说明拆成多个 atomic claims，并评估每个 claim 的可核验价值、类型和优先级。纯观点、提问和过度模糊表达会降低优先级；含主体、动作、时间、数字、官方声明或因果关系的 claim 会提高优先级。
- FIRE 风格动态检索：不预设固定检索深度。系统先执行快速定位；若已找到足够独立强来源且没有冲突，就停止扩展检索；若证据不足、出现反证、涉及高影响 / 学术 / 媒介 claim，则进入标准交叉验证或扩展反证 / 专项渠道。
- InVID 风格媒介工作流：把图片 / 视频验证拆成上下文采集、元数据、关键帧、反向搜索、取证滤镜、AI 检测、地理定位 / 时间线，而不是只给一个总分。

## 2. 处理流程

### 2.1 输入留存

对每次上传建立 `case_id`，保存：

- 原始文本、图片、视频、新闻链接或截图。
- 上传时间、用户提供的上下文、原始文件名。
- 文件哈希、图片尺寸、视频时长、EXIF/metadata、OCR 文本、ASR 转写。
- 用户声称的信息来源，例如“来自 X 截图”“朋友转发”“某媒体报道”。
- 可复核快照，例如网页存档、帖子 URL、平台 ID、截图时间和检索时间。

### 2.2 Claim 抽取

把输入拆成原子化 claim，每条 claim 只表达一个可验证主张：

```text
claim_id: C001
claim_text: 2026 年 4 月 28 日，某公司宣布将在沙特建设一座 200MW AI 数据中心。
claim_type: event
entities: [某公司, 沙特, AI 数据中心]
event_time: 2026-04-28
geography: 沙特
impact_level: high
```

需要分离以下类型：

- 事实：某事件是否发生、某文件是否存在、某人是否说过某句话。
- 数据：价格、金额、人数、市场规模、产能、投票结果。
- 因果：A 导致 B。
- 预测：未来将发生什么。
- 观点：专家判断、媒体评论、KOL 解读。
- 图像 / 视频证据：图片或视频是否真实反映了该事件。

### 2.3 短原子 claim 识别

短句不等于信息量不足。只要用户输入包含明确主体和明确动作，就应当视为可验证的原子 claim，而不是因为字数短直接降级。

例子：

```text
UAE 从 OPEC 退出
OpenAI 收购某公司
某国宣布制裁某企业
某 CEO 辞职
```

处理规则：

- 识别出实体 + 动作后，自动生成检索任务。
- 输出状态应标记为“待联网交叉验证”，而不是“信息量不足”。
- 如果是高影响 claim，在没有完成跨渠道检索前仍可封顶，但封顶原因应是“待联网交叉验证”，不是 claim 本身不可验证。
- 后端接入后，必须自动检索新闻媒体、权威发言、原始文件、互联网平台、自媒体和现实世界旁证。

### 2.4 检索计划

每条 claim 至少执行以下检索：

1. 精确检索：标题、关键句、实体名、数字、日期、地点组合。
2. 多语言检索：原文语言、英文、当地语言、实体别名。
3. 权威来源检索：官方公告、监管文件、法院文件、政府数据库、公司 IR 页面、交易所公告。
4. 新闻检索：通讯社、主流媒体、行业媒体、本地媒体。
5. 社交平台检索：X、Facebook、Instagram、Reddit、微博、YouTube、TikTok、Telegram 等。
6. 来源链检索：最早出现时间、首发账号、首发媒体、转载关系、同源稿件、网页快照。
7. 现实旁证检索：工商、招标、航班、航运、卫星、天气、地震、市场价格、招聘、医院、地方政府通报等。
8. 反向图像 / 视频检索：Google Lens、TinEye、Yandex、Bing Visual Search、关键帧搜索。
9. 时间窗口检索：事件声称时间前后 72 小时；重大事件可扩展到 30 天。
10. 主动证伪检索：为每条 claim 自动生成反向检索式，搜索否认、辟谣、事实核查、更正、撤稿、官方反驳、旧新闻误传和平台标注。

检索结果必须记录 `retrieved_at`，因为网页、帖子和搜索结果会变化。

## 3. 七角度评分模型

### 3.1 默认事件类 claim

```text
最终可信度 =
  联网交叉检索得分 * 22%
  + 逻辑一致性得分 * 15%
  + 历史复盘得分 * 10%
  + 来源链溯源得分 * 15%
  + 现实世界旁证得分 * 15%
  + 统计异常与基准率得分 * 8%
  + 内容与媒介完整性得分 * 15%
```

### 3.2 数据类 claim

如果 claim 的核心是金额、人数、市场规模、投票结果、伤亡数字、产能、价格等数字，应提高统计异常与现实旁证权重：

```text
最终可信度 =
  联网交叉检索得分 * 18%
  + 逻辑一致性得分 * 14%
  + 历史复盘得分 * 10%
  + 来源链溯源得分 * 14%
  + 现实世界旁证得分 * 18%
  + 统计异常与基准率得分 * 20%
  + 内容与媒介完整性得分 * 6%
```

### 3.3 纯声明类 claim

如果验证对象只是“某人是否说过某句话 / 某机构是否发布过某声明”，应提高身份与引用核验、来源链溯源和内容版本权重：

```text
最终可信度 =
  联网交叉检索得分 * 30%
  + 逻辑一致性得分 * 10%
  + 历史复盘得分 * 8%
  + 来源链溯源得分 * 22%
  + 现实世界旁证得分 * 8%
  + 统计异常与基准率得分 * 4%
  + 内容与媒介完整性得分 * 18%
```

### 3.4 分数标签

| 总分 | 结论标签 | 使用规则 |
| --- | --- | --- |
| 90-100% | 已确认 | 可作为强事实引用 |
| 75-89% | 高可信 | 可引用，但需保留主要不确定性 |
| 60-74% | 中等可信 | 可进入分析，但不能单独支撑高影响结论 |
| 45-59% | 证据不足 / 冲突 | 只进入待核查或情景讨论 |
| 25-44% | 低可信 | 不建议采信，只记录为传言 |
| 0-24% | 基本不实 / 已反证 | 不采信，保留反证链 |

## 4. 角度一：联网交叉检索验证

联网交叉检索验证负责回答：公开网络上是否存在可靠、独立、可追溯的证据或反证，并且这些证据是否来自不同类型的渠道。它不只检索新闻网页，还要把新闻媒体、互联网信息平台、自媒体 / KOL、权威发言、原始文件 / 数据库、现实世界旁证、图片 / 视频素材放在同一张证据表里交叉验证。身份与引用核验、传播网络分析并入本角度。

```text
联网交叉检索得分 =
  来源可信度 * 22%
  + 跨渠道覆盖度 * 28%
  + 渠道间一致性 / 冲突 * 20%
  + 身份与引用核验 * 15%
  + 传播网络分析 * 10%
  + 时效与上下文完整度 * 5%
```

### 4.1 跨渠道验证矩阵

| 渠道 | 证据角色 | 默认可信度 | 验证重点 |
| --- | --- | --- | --- |
| 新闻媒体 | 交叉报道、现场报道、专业编辑核验 | 中高 | 是否独立采访、是否同源转载、是否有原始链接 |
| 互联网信息平台 | 传播、目击、现场素材、舆情热度 | 低到中 | 是否多账号多角度、是否认证、是否机器人 / 矩阵扩散 |
| 自媒体 / KOL | 线索、行业解释、早期爆料 | 低到中 | 历史准确率、利益关系、是否提供可核查材料 |
| 权威发言 | 官方声明、当事人声明、机构回应 | 高 | 身份是否真实、原话是否完整、只证明“说过”还是证明事实 |
| 原始文件 / 数据库 | 原始证据、结构化证据 | 很高 | 文件号、发布时间、数据库记录、是否可复核 |
| 现实世界旁证 | 外部痕迹、物理 / 市场 / 机构反应 | 中高 | 招标、航班、卫星、工商、股价、医院、地方通报等是否匹配 |
| 图片 / 视频素材 | 视觉证据、现场证据 | 低到高 | 反向搜索、元数据、地理定位、AI / PS 检测 |

评分规则：

- 至少两个互相独立的渠道一致，才算有效交叉验证。
- 对分析型报道，不能只要求标题或字段完全一致；如果多个独立来源覆盖同一事件链、同一主体、同一时间窗口和同一政策背景，可计为语义交叉支持，但必须标明它验证的是“事件背景 / 逻辑分析”，不是逐字确认原文所有判断。
- 对非英语输入，跨渠道验证必须同时检查英语信息网络中的对应报道和背景证据，例如 Reuters、AP、BBC、Bloomberg、FT、NYT、WSJ、官方英文公告、国际机构数据库、英文事实核查和英文专业分析。
- 英语信息网络证据进入所有角度评分，而不是只进入“联网检索验证”。同一条英文证据可以同时支持来源链、现实旁证、统计基准、内容完整性或主动证伪。
- 上下文深度匹配需要判断：同一主体、同一动作或制度流程、同一时间窗口、同一领域主题、相同数字 / 文件 / 数据口径、以及是否能共同解释原信息中的分析结论。它可以支持“交接面临考验”这类综合判断，但不能替代对具体事实点的原始证据核验。
- 同源转载不算独立证据。标题、核心标题、正文摘要高度相同，或明显来自同一通讯稿 / 同一客户端 / 同一原文的多个页面，必须聚类为一个 story cluster；传播数量只能作为扩散信号，不能叠加事实可信度。
- 用户输入的原始 URL / 原始标题，以及它的同源转载簇，只能作为来源链记录，不能作为外部支持证据、上下文深度匹配加分或交叉报道链接。
- 高影响 claim 最好具备“新闻媒体 / 权威发言 / 原始文件 / 现实旁证”中的至少两个渠道。
- 互联网平台、自媒体、截图和短视频可以提供线索，但如果没有权威发言、新闻媒体或现实旁证支持，必须封顶。
- 现实世界旁证既是独立角度，也可作为联网交叉检索中的一个渠道；区别是本角度看“是否被纳入交叉验证”，角度五看“旁证本身是否硬”。
- 渠道之间出现冲突时，不取平均分，必须列出冲突点并降权。

### 4.2 来源可信度分层

| 层级 | 分数区间 | 典型来源 | 说明 |
| --- | --- | --- | --- |
| T0 | 90-100 | 官方文件、监管公告、法院文件、公司正式公告、原始数据库 | 最适合验证事实、法律状态、财务、政策、制裁 |
| T1 | 80-89 | 通讯社、权威数据库、行业协会、交易所、透明方法论研究 | 适合验证事件、数据、行业结构 |
| T2 | 68-82 | BBC、纽约时报、FT、WSJ、财新等有编辑责任媒体 | 起点较高，但需看是否有原始证据和具名来源 |
| T3 | 50-67 | 垂直媒体、本地媒体、专家访谈、行业播客、企业自媒体 | 可作为线索，通常需要独立证据 |
| T4 | 20-55 | X、微博、Facebook、Instagram、Reddit、论坛、KOL | 弱信号，热度和认证只做有限加分 |
| T5 | 0-25 | 无来源截图、SEO 聚合页、AI 内容农场、无法追溯转载 | 原则上不采信 |

### 4.3 身份与引用核验

检查项：

- 发言人账号、机构网站、媒体采访对象是否真实。
- 是否存在原始采访、全文视频、发布会实录、官方声明或完整上下文。
- 翻译是否准确，是否把条件句、反问、玩笑、转述改成确定陈述。
- 标题中的引述是否和正文、原文语义一致。
- 验证对象是“某人说过”还是“某人说的内容为真”，两者必须分开打分。

封顶规则：

- 只有二手转述，没有原始发言或原始文件：最高 65。
- 只有截图，无法回到原帖或原始采访：最高 35。
- 原话存在但上下文被截断导致语义反转：最高 45。
- 认证账号发布本人声明，可高于 80；但只证明声明存在，不自动证明外部事实。

### 4.4 传播网络分析

检查项：

- 是否为自然扩散，还是多个账号短时间发布相同文案、相同截图、相同链接。
- 是否存在机器人、水军、PR 投放、政治宣传、营销号矩阵或同源搬运。
- 传播是否先由弱信号账号发出，随后被媒体引用，形成“社交媒体 -> 媒体 -> 社交媒体”的循环背书。
- 不同平台上的传播时间线是否一致。
- 高热度是否来自真实讨论，还是少量大号带动的瞬时传播。

评分规则：

- 多平台独立现场信号且时间线一致：+8 到 +15。
- 高点赞、高阅读、高转发：最多 +5，只提高关注度，不替代事实证据。
- 认证名人互动：通常 +3 到 +8，只证明传播和关注度。
- 短时间同文案矩阵扩散：-10 到 -30。
- 明显循环引用或搬运：-8 到 -20。

### 4.5 权威媒体的加减分

正向调整：

- 有原始文件、数据、视频、录音或采访全文链接：+6 到 +10。
- 具名一手信源，例如政府官员、公司发言人、法院文件：+6 到 +10。
- 与至少两个独立来源一致：+8 到 +15。
- 作者长期覆盖该领域且历史准确率高：+3 到 +6。

负向调整：

- 只写“据知情人士透露”“相关人士称”，无进一步证据：-8 到 -15。
- 标题强结论，正文是推测或条件句：-8 到 -12。
- 没有原始链接或关键文件不可访问：-5 到 -10。
- 同一集团、同一通讯社或同一爆料源循环转载：-8 到 -15。
- 与官方文件或硬数据冲突：-25 到 -60。

### 4.6 主动证伪检索

主动证伪检索不是等待普通搜索结果里偶然出现反证，而是为每条 claim 单独生成反方向查询，例如：

```text
<claim> false hoax fake
<claim> denied no plan not leaving
<claim> fact check debunked correction retraction
<claim> 否认 辟谣 更正 撤稿
site:<official-domain> <claim> denied correction
```

规则：

- 反证结果进入独立的 `refuting` 列表，不和支持证据平均。
- 官方否认、监管更正、媒体撤稿、事实核查机构反驳，应触发明显降权。
- 如果反证来自旧时间窗口，必须区分“旧反证已过期”和“当前 claim 被反驳”。
- 如果支持证据和反证同时存在，报告必须显示冲突，不得只给单一结论。

## 5. 角度二：逻辑一致性与反事实检查

逻辑一致性不是主观猜测，而是把 claim 放入同一时期的事件链和现实约束中检验。反事实检查并入本角度：如果该信息是真的，现实中还应该出现哪些迹象；如果这些迹象缺席，是否有合理解释。

```text
逻辑一致性得分 =
  时间线一致性 * 22%
  + 主体动机与利益一致性 * 18%
  + 能力与资源可行性 * 18%
  + 因果链完整性 * 17%
  + 与同期事件兼容性 * 15%
  + 反事实检查 * 10%
```

检查项：

- 时间线：事件声称时间是否早于或晚于相关政策、公告、冲突、财报、选举、发布会。
- 主体动机：当事人是否有动机发布、隐瞒、夸大或制造该信息。
- 能力约束：资金、技术、运输、审批、人员、物理距离是否支持该事件发生。
- 因果链：新闻给出的 A 导致 B 是否有中间环节，还是跳跃式推断。
- 同期事件：是否与同日或同周其他已确认事件相互印证或矛盾。
- 反事实：如果该事件为真，应看到哪些公告、交易、航班、价格、招聘、监管记录或现场迹象；没有看到时是否合理。

扣分规则：

- 关键时间线倒置：-20 到 -50。
- 需要重大资源但无任何审批、资金、物流痕迹：-15 到 -35。
- 因果关系只有叙事，没有证据：-10 到 -25。
- 当事方存在明显宣传、股价、政治或商业激励：-5 到 -20。
- 反事实预期证据大面积缺席且无合理解释：-10 到 -35。
- 与多个已确认事实冲突：-30 到 -70。

## 6. 角度三：历史复盘

历史复盘用于比较相似历史事件、相似误报、相似主体行为和相似来源表现，不能单独推翻强证据。

```text
历史复盘得分 =
  相似事件历史结果 * 35%
  + 同类谣言 / 误报模式匹配 * 25%
  + 来源历史准确率 * 25%
  + 当事方历史行为一致性 * 15%
```

检查项：

- 过去 3-10 年是否出现过高度类似事件。
- 类似事件最终被证实、部分证实还是证伪。
- 该来源过去是否多次抢跑、误报、标题党或准确爆料。
- 相关主体过去是否有夸大、否认后证实、提前泄露、营销炒作的模式。
- 当前事件是否重复了常见谣言模板，例如旧图新传、断章取义、伪造公文、假冒截图。

评分参考：

- 历史上类似事件经常真实发生，且当前证据链更强：75-95。
- 历史上偶有发生，但需要更多确认：55-74。
- 历史上多为误报或宣传，但仍有部分现实基础：35-54。
- 与已知谣言模板高度一致：0-34。

## 7. 角度四：来源链溯源

来源链溯源负责回答：这条信息最早从哪里来，中间经过了哪些转载、改写、翻译和平台扩散。

```text
来源链溯源得分 =
  最早可确认来源 * 35%
  + 原创 / 转载关系清晰度 * 25%
  + 跨平台时间线一致性 * 20%
  + 来源链完整性与可审计性 * 20%
```

检查项：

- 最早可确认发布时间、首发账号、首发媒体或首个文件。
- 后续报道是否独立确认，还是全部引用同一首发源。
- 是否存在网页快照、RSS、编辑记录、平台 ID、归档链接。
- 是否有翻译、摘要、截图、二次剪辑导致含义变化。
- 多个媒体是否只是同一通讯社稿件、同一爆料账号、同一公关稿。

封顶规则：

- 找不到原始来源，只能看到多级转载：最高 55。
- 来源链断裂且关键证据是截图：最高 35。
- 首发源是匿名账号且无后续独立证据：最高 45。
- 找到 T0/T1 原始来源且链路完整：可达 90-100。

## 8. 角度五：现实世界旁证

现实世界旁证负责回答：如果 claim 为真，现实系统中是否留下了可独立验证的痕迹。

```text
现实世界旁证得分 =
  官方 / 结构化记录 * 35%
  + 物理与物流痕迹 * 25%
  + 市场与机构反应 * 20%
  + 本地独立信号 * 20%
```

典型旁证：

- 商业与产业：工商变更、招标、采购、招聘、专利、海关、航运、卫星图、资本开支、土地许可、环评、设备进口。
- 政策与监管：政府公报、监管数据库、法院文件、制裁名单、议会记录、听证会实录。
- 灾害与冲突：气象、地震台网、航班取消、医院通报、地方政府、地图热力、卫星图、现场多角度视频。
- 金融市场：股价、债券、期权、成交量、公告、交易所问询、评级机构动作。
- 社会事件：警情通报、法院记录、学校 / 医院 / 企业内部公告、本地媒体和现场目击材料。

扣分规则：

- 重大事件应有外部记录但完全缺失：-15 到 -40。
- 只有当事方自我声明，没有任何外部痕迹：-10 到 -25。
- 旁证存在但与 claim 的时间、地点、数量不匹配：-15 到 -45。
- 有硬数据侧证且独立于原报道：+10 到 +25。

## 9. 角度六：统计异常与基准率

统计异常与基准率负责回答：该 claim 的数字规模、发生频率、增长幅度和分布位置是否反常，以及反常程度是否需要更强证据。

```text
统计异常与基准率得分 =
  同类事件基准率 * 30%
  + 数值规模合理性 * 25%
  + 分布异常检测 * 20%
  + 口径与样本定义一致性 * 15%
  + 不确定区间表达 * 10%
```

检查项：

- 同类事件过去发生频率是多少，当前 claim 是否落在极端低概率区间。
- 数字是否违反常识上限、产能上限、预算约束、人口规模、物理容量或历史波动范围。
- 增长率、伤亡数、投票率、市场规模、价格波动是否显著偏离历史分布。
- 数据口径是否一致，例如名义 / 实际、含税 / 不含税、日度 / 年度、全球 / 单一区域。
- 是否存在样本选择、幸存者偏差、单位误读、汇率误读、百分比和百分点混淆。

评分参考：

- 数字落在历史合理区间，且有硬数据支持：80-100。
- 数字偏高 / 偏低但有解释变量和侧证：60-79。
- 数字显著异常且证据链普通：35-59。
- 数字几乎不可能或口径明显错误：0-34。

## 10. 角度七：内容与媒介完整性

本角度合并原“图片 / 视频真伪验证”、叙事结构和语言特征、版本演化与删改记录。它负责回答：承载信息的文本、图片、视频、截图或网页本身是否完整、真实、未被误导性加工。

```text
内容与媒介完整性得分 =
  原始素材完整性 * 15%
  + 技术真伪检测 * 25%
  + 旧图 / 旧文 / 旧视频复用排查 * 15%
  + 叙事结构与语言风险 * 20%
  + 版本演化与删改记录 * 20%
  + 界面与格式一致性 * 5%
```

### 10.1 图片 / 视频 / 截图检查

- 文件完整性：是否有原图 / 原视频，还是压缩截图、二次转发文件。
- 元数据：EXIF、拍摄设备、创建时间、GPS、编辑软件痕迹。
- 来源凭证：C2PA / Content Credentials 是否存在、签名是否有效、是否声明 AI 生成或编辑历史。
- 反向搜索：是否为旧图、新裁剪图、镜像图、不同事件图片。
- 视觉一致性：阴影方向、天气、光照、透视、边缘、反射、文字、车牌、路牌。
- 压缩与编辑痕迹：JPEG 量化表、双重压缩、局部噪声异常、重复纹理、边缘羽化、对象粘贴、Error Level Analysis。
- PS / 编辑痕迹：Adobe Photoshop、Lightroom、Canva、GIMP、Snapseed、美图、剪映等软件标记只能证明“经过编辑链”，需要结合视觉异常和来源链判断是否构成造假。
- AI 痕迹：Stable Diffusion、Midjourney、DALL-E、Firefly、ComfyUI、AUTOMATIC1111 等生成工具标记、生成模型水印、手部 / 文字 / 徽标 / 物理结构异常。
- 视频一致性：帧间连续性、音画同步、口型、背景噪声、编码轨迹、关键帧异常。
- 地理定位：建筑、街景、地标、语言、路牌、天气记录、太阳角度。
- 时间验证：天气、新闻直播、赛事、航班、地震、冲突记录是否匹配。

Verité 的自动化媒介取证分为三层：

- 轻量本地预检：读取有限头尾字节，并优先尝试 EXIF/XMP 结构化解析与 C2PA / JUMBF 快速扫描；同时检查 MIME/扩展名一致性、尺寸与压缩率、编辑软件和 AI 生成工具字符串。
- 浏览器取证样本：图片会生成 512px 取证样本；视频会抽取关键帧。系统据此计算轻量 ELA、JPEG Ghost、重采样和时间线一致性代理指标，并在“媒介取证明细”表中展示。
- 可选深度取证接口：本地 `tools/media_ai_service.py` 可接收图片样本和视频关键帧，默认执行 ELA + 频率启发式分析；后续可接入 CLIP-based synthetic detector、UniversalFakeDetect、DIRE、Sherloq、TruFor 等模型或工具，输出 AI detector ensemble 和局部篡改热力图。

若发现 PS / AI 生成等造假风险，必须进入“可疑点”表，而不是只影响隐藏分数。

### 10.2 叙事结构与语言特征

高风险信号：

- 极端情绪化、强煽动性、强敌我叙事。
- 模糊权威来源，例如“外媒爆料”“内部人士称”，但没有媒体名和原始链接。
- 标题非常确定，正文大量使用“或许”“可能”“据传”“尚未确认”。
- 缺少可核查细节，例如时间、地点、主体、文件号、发言原文。
- 大量使用截图代替链接，或用翻译摘要代替原文。

这些信号不能单独判假，但会降低内容完整性，并触发更高验证门槛。

### 10.3 版本演化与删改记录

检查项：

- 媒体是否更改标题、正文、图片、时间戳或作者署名。
- 是否出现 correction、update、editor note、撤稿、删帖、限流或平台标注。
- 网页快照、RSS、社交平台编辑记录是否显示关键事实被修改。
- 初始版本和当前版本之间是否从“确认”改为“据称”、从“死亡”改为“受伤”、从“官方宣布”改为“消息人士称”。

封顶规则：

- 只有截图，无原始链接或原始文件：内容与媒介完整性最高 55。
- 图片 / 视频被反向搜索证实为旧素材新传：最高 25。
- AI / PS 检测强阳性且有视觉异常：最高 35。
- 关键事实被媒体撤回或改写但未说明原因：最高 50。
- 元数据缺失不能直接判假，但会降低原始素材完整性分。
- 原始视频、多角度现场素材、地理定位和时间记录全部一致：可达 85-100。

## 11. 总分封顶与强制降级

无论加权分数是多少，遇到以下情况必须封顶：

- 被 T0 官方文件、法院文件、监管公告直接反驳：最高 20%。
- 关键信息只来自无法追溯截图：最高 35%。
- 高影响 claim 没有独立交叉验证：最高 69%。
- 只有单一社交媒体证据：最高 49%。
- 图片或视频被确认来自其他时间 / 地点 / 事件：最高 30%。
- 来源链存在明显同源循环，且没有原始来源：最高 55%。
- claim 是预测或观点，不是事实：不能标记为“已确认”，最高 74%，除非验证对象只是“某人发表了该观点”。
- 统计数字显著异常但没有更强证据支持：最高 59%。
- 现实世界应有旁证却完全缺席，且无合理解释：最高 60%。

## 12. 后验证复核机制

后验证复核机制是独立机制，不作为评分角度参与单次加权。它负责在事件继续发展时更新结论，避免早期新闻的临时判断变成过期判断。

### 12.1 复核时间表

| 场景 | 复核节点 |
| --- | --- |
| 突发公共事件、战争、灾害、金融风险 | 1 小时、6 小时、24 小时、72 小时、7 天 |
| 政策、监管、制裁、法律事件 | 24 小时、72 小时、7 天、30 天 |
| 企业公告、并购、订单、产品发布 | 24 小时、72 小时、7 天、下一次财报或监管文件 |
| 低影响传闻或舆情 | 24 小时、7 天；若无新证据则归档 |

### 12.2 触发复核的事件

- 官方声明、监管文件、法院文件、公司公告出现。
- 原报道更正、撤稿、删帖、改标题或补充说明。
- 平台添加事实核查标签、删除内容或限制传播。
- 新的原始图片、视频、现场多角度素材、卫星图或本地记录出现。
- 出现强反证，或同一 claim 的关键数字、地点、时间发生变化。
- 事件进入现实世界流程，例如招标、许可、付款、物流、审判、财报披露。

### 12.3 复核输出

每次复核必须记录：

```text
复核时间:
上次结论:
本次结论:
分数变化:
新增证据:
新增反证:
变更原因:
是否需要通知用户:
下一次复核时间:
```

### 12.4 来源与模型学习

复核完成后更新：

- 来源历史准确率。
- 账号 / 媒体的抢跑、误报、撤稿、准确爆料记录。
- 常见谣言模板库。
- 相似事件库。
- 各评分角度的实际命中率，用于校准未来权重。

## 13. 输出格式

每个验证结果必须包含：

```text
结论: 高可信
最终可信度: 82%
适用对象: C001 - 2026 年 4 月 28 日，某公司宣布将在沙特建设一座 200MW AI 数据中心。

分项评分:
- 联网交叉检索: 84% * 22% = 18.5
- 逻辑一致性与反事实检查: 78% * 15% = 11.7
- 历史复盘: 70% * 10% = 7.0
- 来源链溯源: 85% * 15% = 12.8
- 现实世界旁证: 80% * 15% = 12.0
- 统计异常与基准率: 72% * 8% = 5.8
- 内容与媒介完整性: 94% * 15% = 14.1
- 加权总分: 81.9%

能证明其真实的证据 / 逻辑:
1. 公司官网公告和交易所文件均提到该项目，属于 T0 来源。
2. 最早来源为公司官网，后续两家独立媒体报道的地点、投资金额和时间一致。
3. 招标记录和当地许可文件显示项目实体已经进入现实流程。
4. 图片反向搜索未发现旧图，路牌和天气记录与声称地点时间一致。

能证明其虚假的证据 / 逻辑:
1. 投产时间只来自匿名人士，不能视为官方确认。
2. 投资金额在不同媒体间存在 15% 差异，可能口径不同。

可疑点:
1. 官方公告未披露完整投资金额。
2. 上传图片缺少原始 EXIF，不能单独证明拍摄时间。
3. 早期社交平台传播存在同文案搬运。

后验证复核:
1. 24 小时后检查政府许可公告是否更新。
2. 7 天后检查 EPC 招标文件和设备进口记录。
3. 下一次财报后检查公司资本开支指引。
```

## 14. 人机协作建议

最小可用版本：

1. Agent 自动抽取 claim、OCR/ASR、生成检索关键词。
2. Agent 执行联网交叉检索、身份与引用核验、传播网络检查、来源链溯源、现实旁证检索、统计异常检查、内容与媒介完整性检查。
3. Agent 输出七角度评分、证据、反证、可疑点、复核计划。
4. 对高影响 claim、低于 75% 但用户可能采信的信息、涉及人身安全 / 金融 / 法律 / 医疗的内容，强制人工复核。
5. 后验证复核后更新来源历史准确率、谣言模板库、相似事件库和评分校准记录。

建议产品界面：

- Claim List：逐条展示被验证的 claim。
- Evidence Map：展示支持证据、反证和同源转载关系。
- Source Chain Timeline：展示最早来源、转载路径和版本变化。
- Source Rating：展示来源层级和加减分原因。
- Propagation Panel：展示传播网络、异常扩散和认证账号互动。
- Real-world Corroboration Panel：展示官方记录、物理痕迹和市场反应。
- Statistics Panel：展示基准率、异常程度和口径差异。
- Content & Media Integrity Panel：展示图片 / 视频检测、叙事风险和删改记录。
- Score Breakdown：展示七角度百分比和加权贡献。
- Suspicion Panel：集中展示可疑点。
- Post-verification Queue：展示下一次复核时间和触发器。
